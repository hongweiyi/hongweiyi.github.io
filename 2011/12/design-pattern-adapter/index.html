设计模式 - 适配器模式

发表于 2011-12-26   |   分类于 技术分享   |  

GOF**定义：**

“Convert the interface of a class into another interface clients expect. Adapter lets classed work together thar couldn’t otherwise because of incompatible interfaces.”

“将一个类的接口变换成客户端所期待的另一种接口，从而使原本因接口不匹配而无法在一起工作的两个类能够在一起工作。”

国外用的电压和国内的不一样，有110V的330V的，为了让水货电器更好的适配大陆地区，需要额外附加一个适配器。这个和我们所说的适配器模式差不多，都是在不变更对象的情况下，让原本不适合的对象适配起来。

如系统升级的时候，需要考虑到新系统的兼容性，就需要有一个适配器，接受低版本的请求；写多客户端或者多语言的项目的时候，也需要添加适配器，能够处理不同的请求；Android开发中为了让数据能够显示在界面上，也大量用到了适配器模式。

如做一个查询系统，有IBasicInfo以及IEducationInfo两个接口，分别获得用户的基本信息以及教育信息。

  1. package com.wikieno.adapter;  
  2.  
  3. public class BasicInfo implements IBasicInfo {  
  4.  
  5.     /*  6.      获得基本信息  7.      */ 
  6.     @Override 
  7.     public Map<String, String> getBasicInfo() {  
  8.         Map<String, String> map = new HashMap<String, String>();  
  9.         map.put("Name", "Wikie");  
  10.         map.put("Gender", "male");  
  11.         return map;  
  12.     }  

  13. }  

  14. package com.wikieno.adapter;  

  15.  
  16. import java.util.HashMap;  
  17. import java.util.Map;  
  18.  
  19. public class EducationInfo implements IEducationInfo {  
  20.  
  21.     /*  9.      获得教育信息  10.      */ 
  22.     @Override 
  23.     public Map<String, String> getEducationInfo() {  
  24.         Map<String, String> map = new HashMap<String, String>();  
  25.         map.put("Master", "BUAA");  
  26.         map.put("Major", "Software Engineering");  
  27.         return map;  
  28.     }  
  29. }  

使用的时候就比较简单了，

  1. package com.wikieno.adapter;  
  2.  
  3. public class Application {  
  4.     public static void main(String[] args) {  
  5.         IBasicInfo basicInfo = new BasicInfo();  
  6.         IEducationInfo educationInfo = new EducationInfo();  
  7.         System.out.println(basicInfo.getBasicInfo());  
  8.         System.out.println(educationInfo.getEducationInfo());  
  9.     }  
  10. }  

但是后面一拍脑袋，还有工作信息，家庭信息等等等，再继续创建接口与实现类？继续创建吧，但是如果我还想要获得所有信息呢？单独创建每个对象，再单独获得信息，这个就比较麻烦了，不过直接写一个适配器吧！

  1. package com.wikieno.adapter;  
  2.  
  3. public class Application {  
  4.     public static void main(String[] args) {  
  5.         IBasicInfo basicInfo = new BasicInfo();  
  6.         IEducationInfo educationInfo = new EducationInfo();  
  7.         System.out.println(basicInfo.getBasicInfo());  
  8.         System.out.println(educationInfo.getEducationInfo());  
  9.     }  
  10. }  

使用的时候：

  1. package com.wikieno.adapter;   2.   3. public class Application {   4.     public static void main(String[] args) {   5.         IInfoAdapter infoAdapter = new InfoAdapter();   6.         System.out.println(infoAdapter.getInfo());   7.     }   8. }  

最开始的时候我在想：如果设计之初就设计一个IInfo接口，那么什么麻烦都没有。但是设计总是会有纰漏的，适配器模式就是一个“补缺”的工具，用来解决接口不相容的问题。

适配器的优点：

1、 **关联不同的类**

只要你愿意，你就可以让任何两个没有一点关系的类一起运行，只要适配器写得好。

2、 **提高了复用度**

原有的代码还能在原来的系统中用，新增的功能也很好的使用到了旧代码，很好的提高了代码复用度。

3、 **非常灵活**

适配器可以任意添加删除，对系统没有任何影响。
