设计模式 - 抽象工厂模式

发表于 2011-12-16   |   分类于 技术分享   |  

GOF定义：

“Provide an interface for creating familiesof related or dependent objects withoutspecifying their concrete classes.”

“提供一个创建一系列相互依赖对象的接口，而无需指定它们具体的类。”

如果我们需要实例化很多对象，而且这些对象之间有一定的关系；

如果由于需求的变化，需要创建更多系列的对象。

这个时候就可以开始考虑使用抽象工厂模式了。这种模式提供一种“封装机制”来避免客户程序和这种“多系列具体对象创建工作”的紧耦合。

接上一篇工厂的数据库访问例子，原访问接口只有get与set两个方法，但是后面数据库更新了，发现还有一个自增长（incrby）的方法。这下该怎么做呢？修改原来的Command接口，这个不行，违背了开闭原则。既然不能修改原有的代码，那就只能扩展上层的代码了。

那么，在原有的代码基础上，新增Incrable接口：

  1. package com.wikieno.abstractfactory;  
  2.  
  3. public interface Incrable {  
  4.     /*  5.      自增长某个字段  6.      @param args 参数数组  7.      @param step 增长步长  8.      @return 增长后的长度  9.      / 
  5.     public int incrby(String[] args, int step);  
  6. }  

且分别写两个实现类，HashIncrable与SimpleIncrable：

  1. package com.wikieno.abstractfactory;  
  2.  
  3. public class HashIncrable implements Incrable {  
  4.     @Override 
  5.     public int incrby(String[] args, int step) {  
  6.         String key, field;  
  7.         int result = -1;  
  8.         if (2 == args.length) {  
  9.             key = args[0];  
  10.             field = args[1];  
  11.         } else {  
  12.             throw new IllegalArgumentException("参数输入不对");  
  13.         }  
  14.         // 获得client对象，增长相应值  
  15.         // sample: result = conn.incrby…..;  
  16.         return result;  
  17.     }  

  18. }  

  19. package com.wikieno.abstractfactory;  

  20.  
  21. public class SimpleIncrable implements Incrable {  
  22.     @Override 
  23.     public int incrby(String[] args, int step) {  
  24.         String key;  
  25.         int result = -1;  
  26.         if (1 == args.length) {  
  27.             key = args[0];  
  28.         } else {  
  29.             throw new IllegalArgumentException("参数输入不对");  
  30.         }  
  31.         // 获得client对象，增长相应值  
  32.         // sample: result = conn.incrby…..;  
  33.         return result;  
  34.     }  
  35. }  

新增一个AbstactFactory的抽象类：

  1. package com.wikieno.abstractfactory;  
  2.  
  3. /*  4.    5. 抽象工厂类，可生产相应的对象  6.    7. @author 易鸿伟 Wikie  8. @date 2011-12-15  9. */ 
  4. public abstract class AbstractFactory {  
  5.     public abstract Command createCmd();  
  6.     public abstract Incrable createIncrable();  
  7. }  

分别写两个类，实现AbstractFactory抽象类：

  1. package com.wikieno.abstractfactory;  
  2.  
  3. /*  4.    5. Hash工厂，可以创建操作Hash结构的对象  6.    7. @author 易鸿伟 Wikie  8. @date 2011-12-15  9. */ 
  4. public class HashFactory extends AbstractFactory {  
  5.     @Override 
  6.     public Command createCmd() {  
  7.         return new HashCommand();  
  8.     }  
  9.     @Override 
  10.     public Incrable createIncrable() {  
  11.         return new HashIncrable();  
  12.     }  

  13. }  

  14. package com.wikieno.abstractfactory;  

  15.  
  16. /*  4.    5. Simple工厂，可以创建操作Simple结构的对象  6.    7. @author 易鸿伟 Wikie  8. @date 2011-12-15  9. */ 
  17. public class SimpleFactory extends AbstractFactory {  
  18.     @Override 
  19.     public Command createCmd() {  
  20.         return new SimpleCommand();  
  21.     }  
  22.     @Override 
  23.     public Incrable createIncrable() {  
  24.         return new SimpleIncrable();  
  25.     }  
  26. }  

最后就这样使用它：

  1. package com.wikieno.abstractfactory;  
  2.  
  3. public class Application {  
  4.     public static void main(String[] args) {  
  5.         AbstractFactory factory = new HashFactory();  
  6.         Command cmd = factory.createCmd();  
  7.         Incrable incrable = factory.createIncrable();  
  8.         // cmd.get(args);  
  9.         // cmd.set(args);  
  10.         // incrable.incrby(args, step);  
  11.  
  12.         factory = new SimpleFactory();  
  13.         cmd = factory.createCmd();  
  14.         incrable = factory.createIncrable();  
  15.         // cmd.get(args);  
  16.         // cmd.set(args);  
  17.         // incrable.incrby(args, step);  
  18.     }  
  19. }
