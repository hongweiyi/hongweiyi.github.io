设计模式 - 责任链模式

发表于 2011-12-22   |   分类于 技术分享   |  

GOF**定义：**

“Avoid coupling thesender of a request to its receiver by giving more than one object a chance to handle the request. Chain the receiving objects and pass the request along the chain until an object handles it”

“使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合关系。将这些对象连成一条链，并沿着这条链传递该请求，直到有一个对象处理它为止。”

如果你有多个类似的处理逻辑，但是事先不知道到底应该由哪一个处理，你可以考虑使用一下责任链模式。

用户将请求发到责任链的起始对象，如果该对象无法解决问题，则将请求发往链的下一个对象，依此迭代，直到有对象处理请求或者无人处理直接返回。不过在一个纯的责任链模式中，不会出现无人处理的情况，一个请求必须被某一个处理对象所接受。

想象一下腾讯的等级加速情况，普通用户以及VIP1-6（听说快出7了），每个等级都有自己的天数加速策略。我们可以模拟一下，下面这个是加速器抽象类：

  1. package com.wikieno.chainofresponsibility;  
  2.  
  3. public abstract class Speeder {  
  4.     protected Speeder next;  
  5.  
  6.     public void setNext(Speeder next) {  
  7.         Speeder tmp = this;  
  8.         while (null != tmp.next) {  
  9.             tmp = tmp.next;  
  10.         }  
  11.         tmp.next = next;  
  12.         tmp = null;  
  13.     }  
  14.  
  15.     /*  16.      判断是否还有下一个结点  17.         18.      @return  19.      */ 
  16.     public boolean hasNext() {  
  17.         if (null != next) {  
  18.             return true;  
  19.         }  
  20.         return false;  
  21.     }  
  22.  
  23.     public abstract void speed(User user);  
  24.     public abstract boolean canHandle(User user);  
  25. }  

普通用户加速器：

  1. package com.wikieno.chainofresponsibility;  
  2.  
  3. public class SimpleSpeeder extends Speeder {  
  4.  
  5.     @Override 
  6.     public void speed(User user) {  
  7.         if (canHandle(user)) {  
  8.             // TODO  
  9.         } else {  
  10.             if (hasNext()) {  
  11.                 next.speed(user);  
  12.             }  
  13.         }  
  14.     }  
  15.  
  16.     @Override 
  17.     public boolean canHandle(User user) {  
  18.         if (!user.isVIP) {  
  19.             return true;  
  20.         }  
  21.         return false;  
  22.     }  
  23. }  

VIP1加速器：

  1. package com.wikieno.chainofresponsibility;  
  2.  
  3. public class VIP1Speeder extends Speeder {  
  4.  
  5.     public static int LEVEL = 1;  
  6.  
  7.     @Override 
  8.     public void speed(User user) {  
  9.         if (canHandle(user)) {  
  10.             // TODO  
  11.             System.out.println(1+"");  
  12.         } else {  
  13.             if (hasNext()) {  
  14.                 next.speed(user);  
  15.             }  
  16.         }  
  17.     }  
  18.  
  19.     @Override 
  20.     public boolean canHandle(User user) {  
  21.         if (LEVEL == user.levelVIP) {  
  22.             return true;  
  23.         }  
  24.         return false;  
  25.     }  
  26. }  

VIP2同上，就只修改了LEVEL常量。

使用的时候：

  1. package com.wikieno.chainofresponsibility;  
  2.  
  3. public class Application {  
  4.     public static void main(String[] args) {  
  5.         User user = new User();  
  6.         user.isVIP = true;  
  7.         user.levelVIP = 2;  
  8.         Speeder speeder = new SimpleSpeeder();  
  9.         speeder.setNext(new VIP1Speeder());  
  10.         speeder.setNext(new VIP2Speeder());  
  11.           
  12.         speeder.speed(user);  
  13.     }  
  14. }  

这个只是用责任链模式简单模拟了一下QQ等级加速的情况，并不代表现实情况。

显然，VIP6加速就得往下传递6次，有点浪费资源，但是如果腾讯再推出一个VIP7，又有不同的speed策略怎么办呢？用责任链的话，只需要添加一个VIP7Speeder类，在speeder链中加入这个结点即可。在原来的基础之上，不需要修改，只需要在应用中添加一行代码，符合开闭原则。同理，在设计之初，也可以加入删除的接口。
